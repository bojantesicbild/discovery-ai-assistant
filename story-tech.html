ADR-001
MSAL.js v2 over v1 for SSO
Accepted
Better iframe support required for future Outlook add-in context. Silent token refresh works without popup in most cases, critical for embedded scenarios.
Pros
  • Silent refresh in iframes
  • Better error handling API
  • Tree-shakeable modules
Cons
  • Breaking changes from v1
  • Fewer community examples
Sarah Chen • Mar 3, 2026decision.md committed
ADR-002
PostgreSQL over MongoDB
Accepted
Relational model better fits course-enrollment-user relationships. Strong ACID compliance needed for payment flows and grade recording.
Pros
  • ACID transactions
  • Complex joins for reporting
  • Mature ecosystem
Cons
  • Schema migrations needed
  • Less flexible for unstructured data
Sarah Chen • Feb 27, 2026decision.md committed
ADR-003
WebSocket for Real-time Sync
Accepted
Whiteboard requires sub-100ms latency for smooth collaborative drawing. Server-Sent Events too limited for the bidirectional communication needed.
Pros
  • Bidirectional, low latency
  • Native browser support
  • Works with CRDT sync
Cons
  • Connection management complexity
  • Load balancer sticky sessions
Alex Martinez • Mar 1, 2026decision.md committed
ADR-004
React + Vite over Next.js
Accepted
Application is fully client-rendered SPA embedded in Microsoft ecosystem. SSR adds complexity without benefit since all pages require authentication.
Pros
  • Simpler deployment (static files)
  • Faster dev server with Vite
  • Better iframe embedding
Cons
  • No SSR for initial load
  • Manual routing setup
Bojan Tesic • Mar 2, 2026decision.md committed
ADR-005
Y.js CRDT for Whiteboard State
Proposed
Using Y.js as the CRDT library for conflict-free real-time collaboration on the whiteboard. Handles concurrent edits without operational transform complexity.
Pros
  • Conflict-free by design
  • Works offline
  • WebSocket provider built-in
Cons
  • Memory overhead for history
  • Complex debugging
  • Document size can grow
Alex Martinez • Mar 5, 2026Pending review

The Discovery phase (Phase 1) identified NacXwan as a collaborative education platform for the Montenegrin market, designed to run within Microsoft 365. Key outcomes include 23 user stories prioritized by MoSCoW, a validated tech stack, and alignment on the whiteboard-first approach to differentiate from existing LMS tools. This tab captures the decisions and requirements that flow directly into Story & Tech.

Key Requirements Carried Forward
  • 1Microsoft SSO as the only auth method (MSAL.js v2)
  • 2Real-time collaborative whiteboard with sub-100ms latency
  • 3Course catalog with search and category filtering
  • 4Student enrollment with approval workflow
  • 5Video conferencing integration (Teams or Twilio)
  • 6Role-based access: Student, Instructor, Admin
  • 7Embeddable as Outlook add-in for scheduling
  • 8PostgreSQL for relational data persistence
  • 9Responsive design for tablet use in classrooms
  • 10GDPR-compliant data handling for EU users
