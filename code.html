Decision
High Impact
Used MSAL.js v2 with custom token cache strategy
Implemented a custom token cache using sessionStorage instead of the default localStorage approach. This avoids token leakage across tabs and enables per-session auth state. The trade-off is users must re-authenticate when opening a new tab, but this aligns with the security model for an edu platform.

Learning
High Impact
MSAL silent refresh fails in cross-origin iframes
When the React app is embedded in an Outlook add-in iframe, MSAL's acquireTokenSilent fails because third-party cookies are blocked. Workaround: use the redirect flow with a dedicated auth page that posts tokens back via postMessage. This adds complexity but is the only reliable approach for iframe contexts.
Alex M. • Today auth iframe outlook learning.md • Synced
Decision
Medium Impact
Event-driven architecture for notification service
Using an event bus pattern (Redis Pub/Sub) for notifications instead of direct API calls. This decouples the notification service from the core app, making it easier to add new notification channels (email, Teams, push) without modifying the main codebase.
Alex M. • Yesterday architecture notifications decision.md • Synced
Learning
Medium Impact
WebSocket reconnect needs exponential backoff with jitter
Initial implementation used a fixed 1s reconnect interval, which caused a thundering herd when the server restarted during deploy. Switched to exponential backoff (1s, 2s, 4s, 8s, max 30s) with random jitter. Connection recovery went from ~45s average to ~4s after the fix.
Alex M. • 2 days ago websocket reliability learning.md • Synced
Risk
High Impact
Y.js document size grows unbounded without compaction
Y.js CRDT documents retain full edit history by default. For a whiteboard with heavy use, document size could reach several MB within weeks. Need to implement periodic compaction (snapshot + garbage collect) or the WebSocket sync will degrade. No built-in solution — requires custom implementation.
Alex M. • 3 days ago crdt performance risk.md • Synced
Learning
Low Impact
Prisma client generation must run in CI before type-check
TypeScript build was failing in CI because prisma generate wasn't running before tsc. The generated Prisma client types are git-ignored (correct), so CI needs an explicit generate step. Added to the GitHub Actions workflow before the type-check and test steps.
Bojan T. • 3 days ago ci/cd prisma learning.md • Synced
Decision
Medium Impact
Zod for runtime validation at API boundaries
Chose Zod over Joi for request body validation in Express routes. Zod integrates natively with TypeScript (infers types from schemas), reducing duplication between validation logic and type definitions. Applied to all POST/PUT endpoints with a reusable middleware wrapper.
Alex M. • 4 days ago api validation decision.md • Synced
Learning
Low Impact
Vite tree-shaking reduced auth bundle by 40%
Switching MSAL imports from the barrel export to direct module imports (e.g., @azure/msal-browser/dist/auth/PublicClientApplication) allowed Vite to tree-shake unused auth methods. Auth chunk went from 142KB to 86KB gzipped. Worth applying the same pattern to other heavy dependencies.
