Decision
Adopted axe-core for automated accessibility testing
Integrated @axe-core/playwright into the E2E suite to catch WCAG 2.1 AA violations automatically. Runs on every PR against key user flows (login, catalog, enrollment). Catches ~70% of common a11y issues without manual audit.

Learning
MSAL iframe MFA test requires dedicated test tenant
Cannot reliably test MFA-in-iframe scenarios with the shared dev tenant because MFA policies are org-wide. Created a separate Azure AD test tenant with configurable conditional access to isolate SSO integration tests. This unblocked BUG-023 reproduction.
Jana K. • Today auth test-infra learning.md • Synced
Decision
Report Portal over Allure for test reporting
Chose Report Portal (self-hosted) over Allure because it supports real-time test result streaming from CI, has built-in defect classification (product bug vs automation bug vs system issue), and integrates with Jira for auto-linking bugs to failed tests.
Jana K. • Yesterday reporting ci/cd decision.md • Synced
Learning
WebSocket tests need deterministic timing via fake timers
Whiteboard sync tests were flaky because they relied on real setTimeout for reconnect logic. Switching to sinon fake timers made the suite deterministic. Flake rate dropped from ~15% to 0% on CI. Applied the same pattern to all time-dependent WebSocket tests.
Jana K. • 2 days ago websocket flaky-tests learning.md • Synced
Learning
Playwright test isolation requires fresh browser context per spec
E2E login tests leaked session state across specs when sharing a browser context. One test's SSO token polluted another's expected unauthenticated state. Fixed by creating a new browser context (not just a new page) for each test file. Adds ~2s overhead but eliminates all cross-test contamination.
Jana K. • 3 days ago e2e playwright learning.md • Synced
Decision
Minimum 80% coverage gate on all new code
Added a CI coverage gate: PRs fail if new/changed files drop below 80% line coverage. Existing code is exempt (tracked separately). This prevents coverage erosion while not blocking work on legacy areas. Payment module (45%) is flagged for a dedicated coverage sprint.

The QA phase (Phase 4) validates that all code produced in Phase 3 meets the requirements defined in Discovery and Story & Tech. Test coverage targets, traceability from requirements through stories to test cases, and automated regression suites ensure quality gates are met before release. All QA decisions and learnings feed back into the knowledge base.

  • 5WCAG 2.1 AA compliance on all public screens
  • 6Performance: API p95 < 200ms, WebSocket < 100ms
  • 7Automated regression suite runs on every PR
  • 8GDPR: data export & deletion endpoints verified
